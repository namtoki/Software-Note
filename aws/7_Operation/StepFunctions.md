# AWS Step Functions

## Overview
- ワークフローオーケストレーション:       複数サービスを==視覚的なワークフロー==で連携 / ==サーバーレス==
  - 構成:
    - `State Machine`:                  ワークフローの定義 (JSON/YAML)
    - `State`:                          各ステップ (Task, Choice, Parallel, Wait 等)
  - ワークフロータイプ:
    - `Standard`:                       長時間実行 (最大 1 年) / 実行履歴保存 / ==!$0.025/1,000 遷移==
    - `Express`:                        短時間・高頻度 (最大 5 分) / ==!$1.00/100 万リクエスト== / ストリーミング処理向け

---

## 連携サービス

| サービス | 用途 |
|---------|------|
| ==Lambda== | 処理の実行 |
| ==ECS/Fargate== | コンテナタスク実行 |
| ==SNS/SQS== | 通知・キュー連携 |
| ==DynamoDB== | データ操作 |
| ==Batch== | バッチジョブ実行 |

---

## Step Functions vs SWF

| 項目 | ==Step Functions== | ==SWF== |
|------|-------------------|---------|
| **推奨** | ==◎ 推奨== | レガシー |
| **定義** | JSON/YAML | コード |
| **可視化** | ==◎ コンソールで可視化== | なし |
| **用途** | 新規ワークフロー | 既存 SWF アプリ |
