# AWS Migration Hub

移行プロジェクトの==進捗を一元管理==するサービス。

## 特徴
- `一元管理`:                          複数の移行ツールの進捗を==単一コンソール==で追跡
- `統合ツール`:                        DMS, MGN, Application Discovery Service 等と連携
- `アプリケーショングループ`:          関連サーバーをグループ化して管理

---

## AWS Application Discovery Service

オンプレミス環境の==サーバー情報を検出・収集==するサービス。

### 収集方法
| 方法 | 説明 |
|-----|------|
| `Agentless (エージェントレス)` | VMware vCenter 経由 / VM情報を収集 |
| `Agent-based (エージェント)` | サーバーにインストール / ==詳細な情報== (プロセス、ネットワーク依存関係) |

### SAA試験ポイント
- ==オンプレミスサーバーの検出== → Application Discovery Service
- ==依存関係の可視化== → Agent-based

---

## AWS Application Migration Service (MGN)

オンプレミス/他クラウドから AWS への==サーバー移行== (リフト&シフト)。

### 特徴
- `ブロックレベルレプリケーション`:    継続的にデータを複製
- `最小ダウンタイム`:                  カットオーバー時のみ短時間停止
- `自動変換`:                          ブートローダー、ネットワーク等を AWS 向けに変換
- `テスト環境`:                        本番カットオーバー前にテスト可能

### 移行フロー
```
[ソースサーバー] → [継続的レプリケーション] → [テスト] → [カットオーバー] → [AWS]
```

### SAA試験ポイント
- ==サーバー移行 (リフト&シフト)== → MGN
- ==最小ダウンタイム== → MGN (継続的レプリケーション)
- ==SMS== → 非推奨 (MGN を使用)
