# AWS Outposts

- `AWS Outposts`                            ==!高額== / オンプレで AWS を利用できる / AWS が物理的なハードウェアを顧客のデータセンターや施設に設置

## Overview

- **定義**: AWS のインフラ・サービス・API を==オンプレミス==に拡張するフルマネージドサービス
- **特徴**:
  - AWS が顧客サイトにハードウェアを==設置・保守・管理==
  - AWS リージョンと==同じ API・ツール==でオンプレミスにアプリを構築
  - ==シングルテナント== (顧客専用ハードウェア)
  - ==AWS リージョンへの接続が必須==

---

## ユースケース

| シナリオ | Outposts が適切な理由 |
|---------|---------------------|
| ==データレジデンシー== | 法規制でデータをオンプレミスに保持 (HIPAA, GDPR等) |
| ==低レイテンシー要件== | ローカル処理が必要なワークロード |
| ==ハイブリッドクラウド== | オンプレと AWS を同じ API で統合管理 |

---

## Edge サービスの比較 (SAA 頻出)

| 項目 | ==Outposts== | ==Local Zones== | ==Wavelength== |
|------|-------------|-----------------|----------------|
| **設置場所** | ==顧客データセンター== | AWS 所有のメトロ拠点 | 通信事業者のデータセンター |
| **テナント** | ==シングルテナント== | マルチテナント | マルチテナント |
| **主な用途** | オンプレ/ハイブリッド | 都市圏の低レイテンシー | ==5G エッジ==アプリ |
| **コスト** | ==高額== | リージョン+10〜20% | リージョン+20〜30% |

### 選択基準
- **Outposts**: データを==物理的にオンプレミスに保持==する必要がある場合
- **Local Zones**: AWS 管理インフラで都市圏の低レイテンシーが必要な場合
- **Wavelength**: ==5G== デバイス向けの超低レイテンシーアプリ

---

## SAA 出題パターン

1. 「法規制によりデータをオンプレミスに保持しつつ、AWS サービスを利用したい」
   → ==AWS Outposts==

2. 「低レイテンシーが必要だが、自社データセンターは不要」
   → ==Local Zones==

3. 「5G モバイルユーザー向けの超低レイテンシーアプリ」
   → ==Wavelength==

4. 「オンプレの既存システムと AWS を統合し、同じ API で管理したい」
   → ==AWS Outposts==
