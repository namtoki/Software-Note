# AWS Local Zones

- `AWS Local Zones`                         ==!Region より 10〜20% 高い== / リージョンの拡張 / 主要都市や人口密集地域に配置された小規模なデータセンター

## Overview

- **定義**: AWS リージョンを主要都市近くに拡張する==小規模データセンター==
- **特徴**:
  - ==AWS が所有・運用== (顧客サイトではない)
  - ==マルチテナント==
  - ==1桁ミリ秒==の低レイテンシー

---

## ユースケース

| シナリオ | Local Zones が適切な理由 |
|---------|-------------------------|
| ==都市圏の低レイテンシー== | リージョンより近い場所で処理 |
| リアルタイムゲーム | プレイヤー近くにサーバー配置 |
| メディアレンダリング | 低レイテンシーが必要な処理 |

---

## Edge サービスの選択基準 (SAA 頻出)

| シナリオ | 正解 |
|---------|------|
| データを==物理的にオンプレミスに保持==したい | ==Outposts== |
| ==AWS 管理==インフラで都市圏の低レイテンシーが必要 | ==Local Zones== |
| ==5G== デバイス向けの超低レイテンシー | ==Wavelength== |

---

## SAA 出題パターン

1. 「低レイテンシーが必要だが、==自社データセンターは不要==」
   → ==Local Zones==

2. 「特定都市のユーザーに低レイテンシーでサービス提供したい」
   → ==Local Zones==
