# Amazon Route 53

## Overview
- DNS サービス:                           ==!Global== / ==100% SLA== / ==!$0.5/月・HostZone + $0.4/100万クエリ==
  - 機能:                                 ドメイン登録 / DNS ルーティング / ヘルスチェック
  - ホストゾーン:
    - `パブリック`:                       インターネットからのクエリに応答
    - `プライベート`:                     VPC 内からのクエリのみ応答
  - レコードタイプ:
    - `A`:                                ドメイン → ==IPv4==
    - `AAAA`:                             ドメイン → ==IPv6==
    - `CNAME`:                            ドメイン → 別ドメイン (==Zone Apex 不可==)
    - `Alias`:                            ドメイン → ==AWS リソース== (Zone Apex 可、==クエリ無料==)
  - ルーティングポリシー:
    - `Simple`:                           単一リソースへルーティング
    - `Weighted`:                         ==重み付け==で分散 (A/B テスト、段階的移行)
    - `Latency`:                          ==最低レイテンシー==のリージョンへ
    - `Failover`:                         プライマリ障害時に==セカンダリ==へ (DR)
    - `Geolocation`:                      ==ユーザーの地理的位置==でルーティング
    - `Geoproximity`:                     ==リソースの地理的位置==+バイアス
    - `Multivalue Answer`:                複数 IP をランダムに返す (ヘルスチェック付き)
  - ヘルスチェック:                       エンドポイント監視 / 障害時 Failover と連携

## CNAME vs Alias (SAA 頻出)

| 項目 | ==CNAME== | ==Alias== |
|------|----------|-----------|
| **Zone Apex** | ==✕ 不可== | ==◎ 可能== |
| **対象** | 任意のドメイン | ==AWS リソースのみ== |
| **クエリ料金** | 課金 | ==無料== |

※ Zone Apex = ルートドメイン (example.com)

## SAA 出題パターン

| シナリオ | 回答 |
|---------|------|
| 「障害時に別リージョンへ==フェイルオーバー==」 | ==Failover== |
| 「==最もレイテンシーが低い==リージョンへ」 | ==Latency== |
| 「==A/B テスト==で 10% を新バージョンへ」 | ==Weighted== |
| 「ヨーロッパのユーザーは EU リージョンへ」 | ==Geolocation== |
| 「ALB に==ルートドメイン==を向けたい」 | ==Alias レコード== |
