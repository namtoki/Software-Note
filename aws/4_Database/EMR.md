# Amazon EMR (Elastic MapReduce)

## Overview
- ビッグデータ処理:                       ==Hadoop / Spark== 等のマネージドクラスター / ペタバイト級データ処理

---

## サポートフレームワーク

| フレームワーク | 用途 |
|---------------|------|
| ==Apache Spark== | 高速データ処理、ML |
| ==Apache Hadoop== | 分散処理 (MapReduce) |
| ==Presto / Trino== | インタラクティブクエリ |
| ==Hive== | SQL ライクなクエリ |
| ==HBase== | NoSQL データベース |

---

## ノード構成

| ノード | 役割 |
|--------|------|
| **Master** | クラスター管理 |
| **Core** | データ保存 + タスク実行 |
| **Task** | タスク実行のみ (==Spot 推奨==) |

---

## EMR vs 他サービス (SAA 頻出)

| 要件 | サービス |
|------|---------|
| ==Hadoop / Spark== を実行 | ==EMR== |
| SQL でデータ分析 (サーバーレス) | ==Athena== |
| データウェアハウス | ==Redshift== |
| リアルタイムストリーミング | ==Kinesis== |

---

## SAA 出題パターン

| シナリオ | 回答 |
|---------|------|
| 「==Hadoop / Spark== でビッグデータ処理」 | ==EMR== |
| 「ペタバイト級のデータを==分散処理==」 | ==EMR== |
| 「==ETL 処理==を Spark で実行」 | ==EMR== |
| 「コストを抑えてビッグデータ処理」 | ==EMR + Spot インスタンス (Task ノード)== |
