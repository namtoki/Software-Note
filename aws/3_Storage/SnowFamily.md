# AWS Snow Family

==大量データのオフライン転送==と==エッジコンピューティング==を提供するデバイス群。

## デバイス比較

| デバイス | ストレージ | 用途 | エッジ機能 |
|---------|-----------|------|-----------|
| `Snowcone` | ==8TB (HDD) / 14TB (SSD)== | 小規模 / 過酷環境 | EC2, IoT |
| `Snowball Edge Storage` | ==80TB== | 大量データ転送 | EC2, Lambda |
| `Snowball Edge Compute` | ==42TB== | ==エッジ処理==重視 | 高性能 EC2, GPU |
| `Snowmobile` | ==100PB== | 超大量データ | なし |

```
データ量による選択目安:
┌─────────────────────────────────────────────────────┐
│                                                     │
│   〜10TB          →  Snowcone                       │
│   10TB〜80TB      →  Snowball Edge Storage          │
│   80TB〜10PB      →  Snowball Edge (複数台)          │
│   10PB〜          →  Snowmobile                     │
│                                                     │
└─────────────────────────────────────────────────────┘
```

## ユースケース

### データ移行
- ==ネットワーク帯域が不足==している場合の大量データ転送
- データセンター移行
- 災害復旧データのオフサイト保管

### エッジコンピューティング (Snowcone / Snowball Edge)
- ==インターネット接続が限定的==な環境での処理
- 工場、船舶、遠隔地でのデータ処理
- IoT データの前処理

## データ転送の流れ

```
1. AWS コンソールでジョブ作成
        ↓
2. デバイスが届く (==暗号化済み==)
        ↓
3. データをコピー (Snowball Client / OpsHub)
        ↓
4. デバイスを返送
        ↓
5. AWS がデータを S3 にアップロード
        ↓
6. デバイスを==セキュアに消去== (NIST準拠)
```

## セキュリティ
- ==KMS による暗号化== (転送中・保管中)
- 耐タンパー性ケース
- E-Ink 配送ラベル (追跡用)

## OpsHub
- Snow デバイスを管理する==GUIツール==
- ファイル転送、EC2起動、デバイス監視

## SAA 試験ポイント
- ==ネットワーク帯域不足で大量データ移行== → Snow Family
- ==オフライン環境でのデータ処理== → Snowball Edge (エッジコンピューティング)
- ==10PB 以上の移行== → Snowmobile
- ==小規模・過酷環境== → Snowcone
- ==DataSync との使い分け==: ネットワークが十分 → DataSync / 不足 → Snow

## Snow Family vs DataSync

| 要件 | サービス |
|------|---------|
| ネットワーク帯域が十分 | `DataSync` |
| ネットワーク帯域が不足 / 大量データ | `Snow Family` |
| 継続的な同期 | `DataSync` |
| 一回限りの大量移行 | `Snow Family` |
| エッジでのコンピューティング | `Snowball Edge` |
