# AWS DataSync

## Overview
- データ転送サービス:                      オンプレミス ↔ AWS / AWS ↔ AWS 間の高速転送
- フルマネージド:                          ==自動で暗号化・整合性検証・最適化==
- 転送速度:                                ==最大10Gbps== / ネットワーク帯域を効率的に使用

## 対応ストレージ
| ソース/ターゲット | 種類 |
|-----------------|------|
| オンプレミス | NFS, SMB, HDFS, オブジェクトストレージ |
| AWS | ==S3==, ==EFS==, ==FSx== (全種類) |

## Architecture
```
オンプレミス                          AWS
+------------+                    +------------+
|  NFS/SMB   | ← DataSync Agent → |  S3/EFS    |
+------------+      (EC2 or VM)   +------------+
```
- `DataSync Agent`:                        オンプレミスにデプロイ / ==AWS間転送では不要==

## Features
- スケジュール実行:                        定期的な同期ジョブ
- 帯域幅制限:                              業務時間中は制限、夜間はフル活用
- 増分転送:                                ==変更分のみ転送== / 初回以降は高速
- データ検証:                              転送中・転送後に整合性チェック

## Use Cases
- データ移行:                              オンプレミス → S3/EFS への一括移行
- レプリケーション:                        DR用に別リージョンへ定期同期
- コールドデータ移動:                      オンプレミス → S3 Glacier

## DataSync vs 他サービス
| サービス | ユースケース |
|---------|-------------|
| ==DataSync== | 大量データ移行・定期同期 (ネットワーク経由) |
| ==Snowball== | 超大量データ (ネットワーク帯域が不足時) |
| ==Transfer Family== | SFTP/FTPでファイル転送 (既存ワークフロー維持) |
| ==S3 Replication== | S3バケット間のオブジェクトレプリケーション |

## SAA試験ポイント
- 「オンプレミスからS3/EFSへ大量データ移行」→ ==DataSync==
- 「定期的なデータ同期」→ ==DataSync== (スケジュール機能)
- 「ネットワーク帯域が十分」→ ==DataSync== / 「不足」→ ==Snowball==
- 「NFS → EFS 移行」→ ==DataSync==
- AWS間の転送 (例: S3 → FSx) → ==DataSync== (Agent不要)
