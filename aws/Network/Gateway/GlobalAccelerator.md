# AWS Global Accelerator

## Overview
- 高速接続設定:           AWSグローバルネットワークを使用してアプリケーションの可用性とパフォーマンスを向上
  - `静的Anycast IP`:       ==2つの固定IP==を提供 / DNS変更不要
    ```
    【Unicast IP (通常)】
    
      東京ユーザー ─────────────────────┐
                                       │
      米国ユーザー ─────────────────────┼──→ [1つのサーバー 203.0.113.1]
                                       │     (全員がここに接続)
      欧州ユーザー ─────────────────────┘
    
      → 遠いユーザーはレイテンシ大
    
    【Anycast IP (Global Accelerator)】
    
      同じIP 203.0.113.1 が世界中に存在
    
      東京ユーザー ───→ [東京エッジ 203.0.113.1] ───┐
                        (最寄りへ自動)              │
      米国ユーザー ───→ [米国エッジ 203.0.113.1] ───┼──→ [オリジン]
                        (最寄りへ自動)              │    (ALB/NLB/EC2)
      欧州ユーザー ───→ [欧州エッジ 203.0.113.1] ───┘
                        (最寄りへ自動)
    
      → 全員が近いエッジに接続 → 低レイテンシ
    ```
  - `AWSバックボーン`:      インターネットではなく==AWSグローバルネットワーク==経由
  - `エンドポイント`:       ALB, NLB, EC2, Elastic IP
  - `ヘルスチェック`:       エンドポイント障害時に==自動フェイルオーバー==

## ユースケース
- 静的IPが必要なアプリケーション
- 複数リージョンへの高速フェイルオーバー
- ゲーム、IoT、VoIP等のTCP/UDPトラフィック

| 項目 | Global Accelerator | CloudFront |
|-----|-------------------|------------|
| 用途 | ==TCP/UDP== (動的コンテンツ) | ==HTTP/HTTPS== (静的/動的) |
| IP | ==静的IP== (Anycast) | 動的 (DNS) |
| キャッシュ | ==なし== | ==エッジキャッシュ== |
| ユースケース | ゲーム、IoT、VoIP | Webサイト、API、動画配信 |
