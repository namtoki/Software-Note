# AWS Data Pipeline

## Overview
- データ移動・変換:   サービス間でデータを==スケジュール実行==で自動移動 / オンプレ対応
  - 対応:             S3, RDS, DynamoDB, Redshift, ==オンプレ==
  - 処理:             EMR でデータ変換可能

## Data Pipeline vs Glue vs Step Functions

| 項目 | ==Data Pipeline== | ==Glue== | ==Step Functions== |
|------|------------------|----------|-------------------|
| **用途** | データ移動・ETL | ==ETL 特化== | ワークフロー |
| **サーバーレス** | △ (EC2 起動) | ==◎== | ==◎== |
| **オンプレ対応** | ==◎== | △ | △ |
| **推奨度** | レガシー | ==◎ 推奨== | ◎ 推奨 |


## ユースケース

| シナリオ | 回答 |
|---------|------|
| 「==オンプレ==から S3 へ定期的にデータ転送」 | ==Data Pipeline== |
| 「サーバーレスで ETL 処理」 | ==Glue== (Data Pipeline ではない) |
| 「新規で ETL パイプライン構築」 | ==Glue== (推奨) |
