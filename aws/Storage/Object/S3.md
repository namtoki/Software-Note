## Overview
- オブジェクトストレージ:         ==無制限== の容量 / オブジェクトは ==最大5TB== / 耐久性==99.999999999% (11 nines)==
  - `Bucket`:                       ==グローバルで一意== / ==リージョンに作成==
  - `Storage Classes`:
    | クラス | 可用性 | 最小保存期間 | ユースケース |
    |-------|-------|-------------|-------------|
    | `Standard` | 99.99% | なし | 頻繁にアクセス |
    | `Intelligent-Tiering` | 99.9% | なし | アクセスパターン不明 |
    | `Standard-IA` | 99.9% | 30日 | 低頻度アクセス |
    | `One Zone-IA` | 99.5% | 30日 | 低頻度 + 単一AZ (コスト最安) |
    | `Glacier Instant Retrieval` | 99.9% | 90日 | アーカイブ (ミリ秒取得) |
    | `Glacier Flexible Retrieval` | 99.99% | 90日 | アーカイブ (分〜時間) |
    | `Glacier Deep Archive` | 99.99% | 180日 | 長期アーカイブ (12時間以上) |
    - `Glacier Retrieval Options`:
      | オプション | Flexible Retrieval | Deep Archive |
      |-----------|-------------------|--------------|
      | ==Expedited== | 1〜5分 | - |
      | ==Standard== | 3〜5時間 | ==12時間== |
      | ==Bulk== | 5〜12時間 | ==48時間== |

    - `Lifecycle Policy`:           自動移行: ストレージクラス間の自動移行 / 自動削除: 指定日数後に削除
  - 保護:
    - `Versioning`:                 オブジェクトの全バージョンを保持 / 削除保護 (削除しても ==Delete Marker== が付くだけ)
    - `MFA Delete`:                 MFA必須 で削除 (誤削除防止)
    - `Object Lock`:
      - `Governance`:               特権ユーザーは削除/上書き可能
      - `Compliance`:               ==誰も削除/上書き不可== (期間終了まで)
      - `Legal Hold`:               期間なしでロック (訴訟保留等)
  - レプリケーション:             バージョニング有効が必須 (両方のバケット) / 既存オブジェクトは==S3 Batch Replication==で対応
    - `CRR (Cross-Region)`:         別リージョンにレプリケート (DR, レイテンシー低減)
    - `SRR (Same-Region)`:          同一リージョン内でレプリケート (ログ集約, 本番/テスト分離)
  - Encryption:
    - `SSE-S3`:                     S3管理キー (デフォルト, キーローテーション)
    - `SSE-KMS`:                    KMS管理キー (==CloudTrail での監査ログ==, キーローテーション)
    - `SSE-C`:                      顧客提供キー (顧客がキー管理)
    - `CSE`:                        クライアント側でアップロード前に暗号化 
  - Access Control:
    - `Block Public Access`:        パブリックアクセスを一括ブロック (デフォルト有効)
    - `Bucket Policy`:              バケット単位のJSON ポリシー (推奨) /  「このバケットには誰がアクセスできる」か
    - `Access Points`:              大規模アクセス管理を簡素化
    - `Pre-signed URLs`:            ==有効期限付きURL== を生成 / 用途: 認証なしで一時的にオブジェクトを共有
  - 連携:
    - `S3 Event Notifications`:     S3 イベント (PUT, DELETE等) -> Lambda / SQS / SNS / EventBridge
    - `S3 Transfer Acceleration`:   ==!$0.04〜/GB== / ==CloudFront Edge== 経由でアップロード高速化 / 用途: 遠距離からの大容量アップロード
    - `S3 Select / Glacier Select`: ==!有料== / オブジェクト内のデータを ==SQL で抽出== / 全データ取得せず必要部分のみ (コスト/時間削減)
    - `S3 Object Lambda`:           ==取得時に Lambda でデータ変換== / 元データを変更せずに加工して返す
      - ユースケース:             PII マスキング / 画像リサイズ / フォーマット変換 / ユーザーごとに異なるデータ返却
  - `Multipart Upload`:             推奨: ==100MB以上== のファイル / 必須: ==5GB以上== のファイル
  - `CORS`:                         ==別ドメインからのアクセス許可== / 静的サイトホスティングで別ドメインの API 呼び出し時等
