# AWS Trusted Advisor

==AWS環境のベストプラクティス==をチェックし、推奨事項を提供するサービス。

## 5つのカテゴリ

| カテゴリ | チェック内容 |
|---------|-------------|
| `コスト最適化` | 未使用リソース、リザーブド購入推奨 |
| `パフォーマンス` | 過剰プロビジョニング、スループット |
| `セキュリティ` | セキュリティグループ、IAM、MFA |
| `耐障害性` | バックアップ、マルチAZ、冗長性 |
| `サービス制限` | ==クォータの使用率== (80%超で警告) |

## サポートプランによる制限

| サポートプラン | 利用可能なチェック |
|---------------|-------------------|
| `Basic / Developer` | ==7つの基本チェックのみ== (セキュリティ系) |
| `Business 以上` | ==全チェック== + API アクセス |

### Basic/Developer で利用可能なチェック (7つ)
- S3 バケットのパブリックアクセス
- セキュリティグループ (無制限アクセス)
- IAM の使用状況
- ルートアカウントの MFA
- EBS パブリックスナップショット
- RDS パブリックスナップショット
- サービス制限 (一部)

## Organizations 統合
- ==委任管理者==を設定して組織全体のチェック結果を集約
- 組織ビューで全アカウントの推奨事項を確認

## SAA 試験ポイント
- ==ベストプラクティスのチェック== → Trusted Advisor
- ==サービスクォータの監視== → Trusted Advisor (サービス制限カテゴリ)
- ==全機能利用には Business サポート以上==が必要
- ==セキュリティグループの設定ミス検出== → Trusted Advisor

## Trusted Advisor vs 他サービス

| 要件 | サービス |
|------|---------|
| 全般的なベストプラクティス | `Trusted Advisor` |
| EC2/Lambda の詳細な最適化 (ML分析) | `Compute Optimizer` |
| セキュリティアラートの集約 | `Security Hub` |
| クォータの引き上げ申請 | `Service Quotas` |
| 設定変更の追跡・コンプライアンス | `Config` |

## SAA 出題パターン

| シナリオ | 回答 |
|---------|------|
| 「AWS環境の==ベストプラクティス==をチェック」 | Trusted Advisor |
| 「==サービス制限==に近づいているか確認」 | Trusted Advisor |
| 「==セキュリティグループ==が 0.0.0.0/0 になっていないか」 | Trusted Advisor |
| 「==コスト削減==の推奨事項を確認」(Business以上) | Trusted Advisor |
| 「EC2を==機械学習==で分析して最適化」 | Compute Optimizer |


- ==!AWS Trusted Advisor==                     ==!Global== / ==Best Practice== 基づき分析改善提案 / 不使用インスタンス /  ==SG チェック==
  - `セキュリティー`                        Basic / AWSのセキュリティー関連機能が有効化されているかどうか、アクセスキーが流出していないかなど、セキュリティーに関するチェック
  - `サービス制限（クォーター）`            Basic / AWSアカウントが作成できるストレージやロールなどのリソースの最大数を監視し、80％を超えた場合にはアラートを発出
  - `コスト最適化`                          Business / サービスや設定の使用状態をチェックし、活用されていないサービスを洗い出してコスト最適化の提案
  - `パフォーマンス`                        Business / AWSリソースのパフォーマンス向上のため、スループットや有効化すべき設定をチェック
  - `耐障害性`                              Business / 耐障害性、可用性や信頼性を向上させるため、オートスケーリングやヘルスチェック、その他有効／無効にすべき設定をチェック
  - `運用上の優秀性`                        Business / AWS上のサービスが安定して稼働していること、問題発生時に迅速な対応を行える状態であることを確認
