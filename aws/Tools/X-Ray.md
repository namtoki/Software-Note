# AWS X-Ray

## Associate Level
- 分散トレーシング:                       ==リクエストの流れを可視化== / パフォーマンス分析 / ボトルネック特定

---

## 概念

```
ユーザー → API GW → Lambda → DynamoDB
              ↓
         X-Ray がトレース
              ↓
    ┌─────────────────────────┐
    │    サービスマップ        │
    │  API GW ─→ Lambda ─→ DB │
    │   20ms     150ms    30ms │ ← どこが遅いか可視化
    └─────────────────────────┘
```

---

## 主な機能

| 機能 | 説明 |
|------|------|
| ==サービスマップ== | リクエストの流れを可視化 |
| ==トレース== | 個々のリクエストを追跡 |
| ==分析== | レイテンシー、エラー率の特定 |

---

## 対応サービス

| サービス |
|---------|
| Lambda, API Gateway |
| EC2, ECS, EKS |
| Elastic Beanstalk |
| SNS, SQS |

---

## CloudWatch vs X-Ray

| 項目 | ==CloudWatch== | ==X-Ray== |
|------|---------------|-----------|
| **用途** | メトリクス・ログ監視 | ==分散トレーシング== |
| **対象** | 個々のサービス | ==サービス間の流れ== |
| **分析** | リソース使用率 | ==リクエスト追跡== |
