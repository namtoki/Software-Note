# AWS Resource Access Manager (RAM)

## Overview
- リソース共有:                           ==マルチアカウント==間でリソースを共有 / リソースの重複作成を回避

---

## 共有可能なリソース例

| リソース | ユースケース |
|---------|-------------|
| ==VPC サブネット== | 複数アカウントで同一 VPC を使用 |
| ==Transit Gateway== | ネットワークの一元管理 |
| Route 53 Resolver | DNS 共有 |
| License Manager | ライセンス共有 |

---

## VPC 共有の例

```
┌─────────────────────────────────────────┐
│          Owner Account                  │
│  ┌─────────────────────────────────┐   │
│  │        VPC (共有)               │   │
│  │  ┌───────────┐ ┌───────────┐    │   │
│  │  │ Subnet A  │ │ Subnet B  │    │   │
│  │  └───────────┘ └───────────┘    │   │
│  └─────────────────────────────────┘   │
└─────────────────────────────────────────┘
        ↓ RAM で共有
┌─────────────────┐ ┌─────────────────┐
│  Account A      │ │  Account B      │
│  Subnet A 利用  │ │  Subnet B 利用  │
└─────────────────┘ └─────────────────┘
```

---

## SAA 出題パターン

| シナリオ | 回答 |
|---------|------|
| 「複数アカウントで==VPC を共有==したい」 | ==RAM== |
| 「==マルチアカウント==でリソースを共有」 | ==RAM== |
| 「リソースの==重複作成を避け==たい」 | ==RAM== |
| 「Transit Gateway を複数アカウントで共有」 | ==RAM== |
