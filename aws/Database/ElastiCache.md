# Amazon ElastiCache

==インメモリ==キャッシュサービス。ミリ秒以下のレイテンシを実現。

## エンジン比較 (SAA頻出)

| 項目 | Redis | Memcached |
|-----|-------|-----------|
| データ構造 | ==複雑== (文字列, リスト, セット, ハッシュ等) | シンプル (文字列のみ) |
| 永続性 | ==あり== (スナップショット, AOF) | ==なし== |
| レプリケーション | ==あり== (Multi-AZ) | なし |
| クラスターモード | あり (シャーディング) | あり |
| バックアップ | ==あり== | なし |
| ユースケース | セッション管理, リーダーボード, Pub/Sub | 単純なキャッシュ |

## Redis
- `Multi-AZ`:                          ==自動フェイルオーバー== / 高可用性
- `リードレプリカ`:                    最大5台 / 読み取りスケーリング
- `クラスターモード`:                  ==シャーディング== / 書き込みスケーリング
- `バックアップ/復元`:                 スナップショット対応
- `暗号化`:                            転送中 (TLS) / 保存時 (KMS)

## Memcached
- `マルチスレッド`:                    シンプルなキャッシュに最適
- `スケーリング`:                      ノード追加/削除で水平スケール
- `データ永続性なし`:                  ノード障害でデータ消失

## キャッシュ戦略
- `Lazy Loading`:                      キャッシュミス時にDBから取得 → キャッシュに保存
- `Write Through`:                     DB書き込み時にキャッシュも更新
- `TTL`:                               有効期限設定でデータ鮮度を管理

## SAA試験ポイント
- ==永続性 / レプリケーション / バックアップ== → Redis
- ==シンプルなキャッシュ / マルチスレッド== → Memcached
- ==セッション管理 / リーダーボード== → Redis
- ==高可用性 (Multi-AZ)== → Redis
- ==DBの負荷軽減== → ElastiCache (Read負荷をオフロード)
